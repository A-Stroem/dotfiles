## Key Decisions / Content

### Hostname Convention
- Format: `<role>-<number>` (e.g. `nas-01`, `pi-01`, `sw-01`)
- Hostnames identify the machine, not its services — avoids hostnames becoming lies when software moves
- Enterprise format (`<country><location><device><number>`) is overkill for homelab scale
- Exception: short descriptor allowed when multiple same-role devices serve genuinely distinct purposes

### IP Plan Structure
- 5 VLANs: mgmt (1), servers (10), trusted (20), iot (30), guest (40)
- Suggested `10.0.x.0/24` scheme per VLAN
- WiFi SSIDs mapped to VLANs: `home` → trusted, `iot` → iot, `guest` → guest
- `mgmt` and `servers` VLANs are wired-only (no SSID)
- Reserved range structure: `.1` gateway, `.2–.9` infra, `.10–.49` static, `.100–.199` DHCP

### 1Password Naming
- Format: `<project/host> — <credential type>` using em dash separator
- Device creds: `nas-01 — Root Password`, `pi-01 — SSH Key`
- Service creds: `netbox — DB Password`, `grafana — Admin`
- Network creds: `network — home WiFi`, `network — Router Admin`
- Recommend dedicated Homelab vault separate from personal/work
