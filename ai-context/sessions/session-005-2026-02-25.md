## Architecture Decisions

- **Netbox** for CMDB (physical + virtual inventory, IP management, drive bay tracking)
- **Prometheus + Grafana + node-exporter + smartctl-exporter** for monitoring (Phase 2, on NAS)
- **Phased deployment**: Netbox on Mac now ‚Üí full stack on NAS when built
- **Docker Compose** (not Kubernetes) ‚Äî matches existing Pi infrastructure pattern
- Netbox gets its own dedicated Postgres + 2x Redis (not shared with other apps)
- nginx routing follows same pattern as Pi's `docker-infra` (variable-based proxy_pass)

## Bugs Found and Fixed (important for NAS re-deployment)

| Bug | Root cause | Fix |
|---|---|---|
| Netbox wouldn't start | `SECRET_KEY` only 30 chars, Django requires ‚â•50 | Regenerate with `python3 -c "import secrets; print(secrets.token_hex(50))"` (100 chars) |
| Healthcheck failing | `/api/` returns 403 in Netbox v4.x (auth required) | Use `/login/` as healthcheck endpoint |
| Housekeeping crashing (exit 127) | `/opt/netbox/housekeeping.sh` removed in v4.x | Use `manage.py housekeeping` in `while true; sleep 86400` loop |
| Env vars not loaded in containers | direnv uses `export VAR=value`, not picked up by subagent shells | User must run `docker compose up` from direnv-active terminal |
